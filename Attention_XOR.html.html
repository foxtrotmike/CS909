XOR attention inspector

This demo shows what self-attention is actually doing by applying it to the XOR classification problem. Each input, [0,0], [0,1], [1,0], or [1,1], is treated as a set of two scalar tokens, one per feature. The model first maps each token through learnable linear query, key, and value projections. It then computes a 2 × 2 attention matrix that measures how strongly each token attends to itself and to the other token within the same input. These attention weights are used to combine the value vectors into new token representations in a hidden space. The attended representations are then flattened and passed to either a linear readout or a nonlinear fully connected readout to produce the final prediction. Along the way, the demo displays the query, key, value, score, attention, and attended output tensors so you can see exactly how the learned representation is constructed and how the final classifier uses it.

Token projections
Q = x × Wq, K = x × Wk, V = x × Wv
Attention
Scores = QKT / √dqk, A = softmax(Scores), H = AV
Readout options
Linear: ŷ = wT tanh(flatten(H)) + b
Nonlinear: ŷ = W2 tanh(W1 tanh(flatten(H)) + b1) + b2

Targets are -1 for equal bits and +1 for different bits.
